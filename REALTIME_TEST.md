# Тестирование Real-time функций

## 🧪 Как проверить что сообщения обновляются мгновенно

### Подготовка

1. Запустите приложение:
```bash
docker-compose up -d
```

2. Откройте два окна браузера (или два разных браузера):
   - Окно 1: `http://localhost`
   - Окно 2: `http://localhost` (в режиме инкогнито или другой браузер)

### Тест 1: Регистрация двух пользователей

**Окно 1:**
1. Зарегистрируйтесь как `user1@test.com`
2. Имя: `User One`

**Окно 2:**
1. Зарегистрируйтесь как `user2@test.com`
2. Имя: `User Two`

### Тест 2: Создание чата

**Окно 1:**
1. Нажмите кнопку "+" (Новый чат)
2. Введите `user2@test.com` в поиск
3. Выберите User Two из результатов
4. Чат будет создан

**Окно 2:**
1. Обновите страницу или подождите 30 секунд
2. Новый чат должен появиться в списке

### Тест 3: Мгновенная отправка сообщений

**Окно 1:**
1. Откройте чат с User Two
2. Напишите: "Привет! Это тест real-time"
3. Нажмите Enter

**Окно 2:**
1. Откройте чат с User One
2. Сообщение должно появиться МГНОВЕННО (< 100ms)
3. Без перезагрузки страницы

**Окно 2:**
1. Напишите ответ: "Получил! Работает отлично"
2. Нажмите Enter

**Окно 1:**
1. Ответ должен появиться МГНОВЕННО

### Тест 4: Индикатор "печатает..."

**Окно 1:**
1. Начните печатать сообщение (не отправляйте)

**Окно 2:**
1. Должен появиться статус "печатает..." под именем чата
2. Статус исчезнет через 3 секунды после остановки печати

### Тест 5: Статусы online/offline

**Окно 1:**
1. Проверьте статус User Two в заголовке чата
2. Должно быть "онлайн"

**Окно 2:**
1. Закройте вкладку или выйдите из системы

**Окно 1:**
1. Статус должен измениться на "офлайн" МГНОВЕННО

### Тест 6: Отправка файлов

**Окно 1:**
1. Нажмите кнопку 📎
2. Выберите изображение
3. Файл отправится

**Окно 2:**
1. Изображение должно появиться МГНОВЕННО
2. Можно кликнуть для просмотра

### Тест 7: Голосовые сообщения

**Окно 1:**
1. Нажмите кнопку 🎤
2. Разрешите доступ к микрофону
3. Запишите короткое сообщение
4. Нажмите "Отправить"

**Окно 2:**
1. Голосовое сообщение появится МГНОВЕННО
2. Можно воспроизвести

### Тест 8: Множественные сообщения

**Окно 1:**
1. Быстро отправьте 5 сообщений подряд:
   - "Сообщение 1"
   - "Сообщение 2"
   - "Сообщение 3"
   - "Сообщение 4"
   - "Сообщение 5"

**Окно 2:**
1. Все 5 сообщений должны появиться МГНОВЕННО
2. В правильном порядке
3. Без дубликатов

### Тест 9: Уведомления

**Окно 2:**
1. Откройте другой чат (не с User One)

**Окно 1:**
1. Отправьте сообщение "Тест уведомлений"

**Окно 2:**
1. Должно появиться уведомление в правом верхнем углу
2. С текстом сообщения

### Тест 10: Переподключение

**Окно 1:**
1. Откройте DevTools (F12)
2. Перейдите на вкладку Network
3. Найдите WebSocket соединение
4. Закройте его (правый клик → Close)

**Результат:**
1. Через несколько секунд соединение восстановится автоматически
2. Появится уведомление "Подключено к серверу"
3. Все функции продолжат работать

## 🐛 Возможные проблемы

### Сообщения не появляются мгновенно

**Проверьте:**
1. WebSocket подключен (в DevTools → Network → WS)
2. Backend логи показывают отправку
3. Frontend логи показывают получение
4. Нет ошибок в консоли

**Решение:**
```bash
# Перезапустите backend
docker-compose restart backend

# Очистите кэш браузера
Ctrl+Shift+Delete → Clear cache
```

### Дубликаты сообщений

**Причина:** Сообщение приходит и через API и через WebSocket

**Проверьте:** В коде есть проверка на дубликаты:
```javascript
const exists = appData.messages[chatId].find(m => m.id === message.id);
if (exists) return;
```

### WebSocket не подключается

**Проверьте:**
1. Nginx настроен для WebSocket
2. Порт 80 открыт
3. JWT токен валидный

**Решение:**
```bash
# Проверьте Nginx
docker-compose logs nginx

# Проверьте Backend
docker-compose logs backend
```

### Нагрузочное тестирование

Для тестирования под нагрузкой используйте:
```bash
# Установите wscat
npm install -g wscat

# Подключитесь
wscat -c "ws://localhost/ws?token=YOUR_TOKEN"

# Отправьте сообщение
{"type":"typing","chatId":1}
```
